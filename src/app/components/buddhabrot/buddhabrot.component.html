Guide

How it behaves

  • Start repeatedly requests random samples so the image gradually resolves.
  • Single batch runs one accumulation cycle without entering the loop.
  • Clear resets the accumulator without changing the current view window.
  • Reset view restores the default frame and clears the current accumulation.
Mechanics

What's actually happening

Unlike the Mandelbrot render, this doesn't compute one iteration count per pixel. Instead, each batch throws thousands of random points c at the current view window and traces the same escape trajectory, recording every pixel it passes through along the way:

z = z² + c

  • If a point escapes before the iteration cap — meaning it's outside the Mandelbrot set — every pixel its path touched gets a "hit" added to an accumulator.
  • Points that never escape (the set itself) are discarded entirely and contribute nothing to the image.
  • Batches alternate contributing to the red, green, and blue channels in turn, so the color you see is built from three separately accumulated passes.
  • Each redraw normalizes every channel against its own brightest pixel and applies a gamma curve, keeping faint, low-density areas visible next to the brightest ones.

That's why the image looks ghostly and takes time to resolve: it's built entirely from the paths of escaping points, and more samples mean a cleaner accumulation of where those paths tend to go.
